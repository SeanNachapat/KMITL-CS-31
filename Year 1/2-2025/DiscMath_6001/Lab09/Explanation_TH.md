# คำอธิบายโปรแกรมตรวจสอบคุณสมบัติของความสัมพันธ์ (Relation Checker)

โปรแกรมนี้ประกอบด้วยไฟล์ Java 3 ไฟล์ที่ทำงานร่วมกันเพื่อรับค่าเซต (Set) และความสัมพันธ์ (Relation) จากนั้นทำการตรวจสอบคุณสมบัติทางคณิตศาสตร์ 3 ประการ ได้แก่ สะท้อน (Reflexive), สมมาตร (Symmetric) และ ถ่ายทอด (Transitive)

---

## 1. `Set.java`
ไฟล์นี้ใช้สำหรับจัดการข้อมูลของ **เซต (Set)**
- **ตัวแปร `set`**: เก็บตัวเลขจำนวนเต็ม (Integer) ในรูปแบบของ `ArrayList`
- **ฟังก์ชัน `add(int value)`**: เพิ่มสมาชิกใหม่เข้าในเซต โดยจะตรวจสอบก่อนว่ามีค่านั้นอยู่แล้วหรือไม่ (เพื่อรักษาคุณสมบัติของเซตที่สมาชิกต้องไม่ซ้ำกัน)
- **ฟังก์ชัน `contains(int value)`**: ตรวจสอบว่ามีตัวเลขที่ระบุอยู่ในเซตหรือไม่
- **ฟังก์ชัน `getElements()`**: ส่งคืนรายการสมาชิกทั้งหมดในเซต

## 2. `Relation.java`
ไฟล์นี้เป็นส่วนหัวใจของโปรแกรม ใช้สำหรับเก็บข้อมูล **ความสัมพันธ์ (Relation)** และตรวจสอบคุณสมบัติต่างๆ
- **Inner Class `Pair`**: คลาสย่อยที่ใช้เก็บคู่ลำดับ $(x, y)$
- **คุณสมบัติที่ตรวจสอบ**:
    - **Reflexive (สะท้อน)**: ตรวจสอบว่าสำหรับทุกสมาชิก $a$ ในเซต จะต้องมีคู่ลำดับ $(a, a)$ อยู่ในความสัมพันธ์
    - **Symmetric (สมมาตร)**: ตรวจสอบว่าถ้ามีคู่ลำดับ $(a, b)$ จะต้องมีคู่ลำดับ $(b, a)$ อยู่ในความสัมพันธ์ด้วย
    - **Transitive (ถ่ายทอด)**: ตรวจสอบว่าถ้ามีคู่ลำดับ $(a, b)$ และ $(b, c)$ แล้ว จะต้องมีคู่ลำดับ $(a, c)$ อยู่ในความสัมพันธ์
- **ฟังก์ชัน `checkAll()`**: แสดงคู่ลำดับทั้งหมดและผลการตรวจสอบคุณสมบัติทั้ง 3 ประการ

## 3. `Main.java`
ไฟล์หลักที่ใช้รันโปรแกรมและรับข้อมูลจากผู้ใช้ผ่าน **Scanner**
- **ส่วนการรับค่า Set**: รับข้อมูลสมาชิกของเซตเป็นบรรทัดเดียว (เช่น `0 1 2 3`)
- **ส่วนการรับค่า Relation**: 
    1. รับจำนวนคู่ลำดับ (Number of pairs)
    2. วนลูปรับค่าคู่ลำดับแต่ละคู่ (เช่น `0 1`) จนครบจำนวนที่ระบุ
- **การประมวลผล**: เรียกใช้ `relation.checkAll()` เพื่อแสดงผลลัพธ์การตรวจสอบ

---

### วิธีการใช้งาน (Example)
1. ระบุสมาชิกของเซต: `Set: 0 1 2`
2. ระบุจำนวนคู่ลำดับ: `Relation: 2`
3. ใส่คู่ลำดับ:
   - `Enter pair 1: 0 0`
   - `Enter pair 2: 1 1`
4. โปรแกรมจะแสดงผลลัพธ์ว่าความสัมพันธ์นี้มีคุณสมบัติใดบ้าง
